# LTD Capability Mesh Operating Model

Chummer owns product truth, story truth, package truth, release truth, support truth, rules truth, and identity truth.

LTDs are bounded adapters for:

- capture
- draft
- projection
- notification
- review
- audit

Authority boundaries:

- Product signals stay first-party in Chummer and may mirror to a hosted board.
- Dispatches stay receipt-backed and may use external draft assistance, but drafts never publish directly.
- Projection tables are operator mirrors only and never feed runtime truth back into Chummer.
- Voice capture becomes a structured EA packet before any downstream repo action.

Current mesh:

- `ProductLift.dev`: public signal mirror only
- `Teable`: operator projection only
- `Syllabbles`: dispatch draft only
- `blipai.app`: operator voice capture only
- `Emailit`: delivery adapter only
- `1min.AI`: bounded synthesis fallback
- `BrowserAct`: verification and account-fact capture

Required proof:

1. Chummer creates the first receipt.
2. The LTD adapter creates a projection or draft receipt.
3. Public copy never exposes provider names or secret-bearing internals.
4. Private support and campaign data never enter public or draft lanes.
